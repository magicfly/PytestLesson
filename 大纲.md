## 第一课 Pytest介绍
1. 什么是 Pytest？
Pytest 是一个功能强大且易于使用的 Python 测试框架。它能够支持简单的单元测试（Unit Testing），也能扩展到复杂的功能测试（Functional Testing）。

- 开源性质：由社区驱动，拥有庞大的插件生态系统。

- 兼容性：它不仅能运行自己编写的用例，还能无缝兼容 Python 自带的 unittest。

- 设计理念：提倡用最少的代码做最多的事（Minimalistic style）。

1. Pytest 的核心优点

- 入门简单：不需要像 unittest 那样必须写类（Class）。只需编写以 test_ 开头的函数即可运行。

- 强大的断言（Assert）：直接使用 Python 原生的 assert 表达式（如 assert a == b）。出错时，Pytest 会自动提供非常详细的变量值对比，无需记忆 assertEqual 等复杂方法。

- Fixture 依赖注入系统：这是 Pytest 的杀手锏。它比传统的 setup/teardown 更灵活、可复用。可以轻松管理数据库连接、登录态等测试前置条件。

- 参数化测试（Parametrization）：用一个装饰器 @pytest.mark.parametrize 就能实现一组代码跑多组数据，极大减少重复代码。

- 生态丰富：拥有上千个插件（如 pytest-html 生成报告、pytest-xdist 实现并行测试）。

## 第三课 运行Pytest, 测试用例管理
- 运行pytest, pytest.main()
- 测试用例命名
    - 测试模块：默认以test_开头或者_test.py结束。
    - 测试类：必须以Test开头命名，且测试类中不能有 init 方法。
    - 测试方法/测试函数：必须以test开头，如：test_login() 或 testRegister()
    - pytest 框架可以通过pytest.ini配置文件自定义命名规则
- 测试用例执行顺序
    - 模块默认根据名称的字母逐一比较，根据ASCII值，越小越先。
    - 测试类与测试方法默认按照从上到下顺序执行
    - 可以通过插件改变执行顺序, pip install pytest-ordering
- 测试用例断言
    - 验证预期结果与实际结果是否一致
    - 函数返回值断言
    - 表达式断言
    - 比较类型断言
    - 数据类型比较断言
    - 触发指定异常的断言
    - 为断言添加说明
